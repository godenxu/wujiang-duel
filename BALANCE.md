# 战斗数值与公式说明（BALANCE）

> 本文档汇总单挑（轮换出招回合制）的全部数值与公式，便于查阅与调整。
> 变量缩写：体=体力 · 武=武力 · 统=统帅 · 智=智力 · 政=政治 · 魅=魅力；
> 「己」=出招方，「敌/守」=承受方。实现见 `js/engine.js`，评级见 `js/app.js`。

## 一、体力 / 战意

- **体力上限 = 初始体力 = `体`**，归零即 KO。
- **战意初始** = `min(100, round(55 + 政 × 0.45))`
- **每回合末回战意** = `2 + 政 / 22`
- **出招战意消耗** = `max(2, round(招式基础耗 × (1 − min(0.34, 政/300))))`；基础耗为 0 则不耗。

| 招式 | 猛攻 | 普攻 | 格挡 | 谋攻 | 束缚 | 弱化 | 疗伤 | 蓄力 |
|---|---|---|---|---|---|---|---|---|
| 基础耗 | 20 | 0 | 0 | 7 | 12 | 10 | 11 | 4* |

\* 蓄力实际是「恢复战意」，并不扣除。

## 二、先手 / 回合

- **先手**：`spd = 统 + 随机(0~20)`，高者先手；之后双方**逐回合交替出招**。
- 被束缚的一方轮到自己时**跳过该回合**（消耗一层束缚）。

## 三、物理攻击伤害（猛攻 / 普攻）

```
base       = 武                              （只看攻方武力）
mitigation = (1 − min(0.50, 守.统/380))      （守方统帅减伤）
           × (1 − min(0.12, 守.政/1000))     （守方政治微韧性）
counter    = 相克表[招式][守方姿态]
critMul    = 蓄势命中 ? 2.0 : 1.0            （蓄力「暴发」翻倍）
luck       = 随机 0.82 ~ 1.18
atkMul     = 被「弱化」时 <1，否则 1
power      = 猛攻 1.0 / 普攻 0.5

伤害 = base × 0.32 × counter × mitigation × critMul × luck × atkMul × power
```

随后叠加**暴击**（见第五节），最后 `dmg = max(1, round(dmg))`。

## 四、谋攻伤害（智力系，无视格挡）

```
成功率 = clamp(0.55 + (己.智 − 守.智)/220, 0.12, 0.94)
  失败 → 未命中（0 伤害，"被识破"）
  成功 ↓
base       = 己.智
mitigation = 1 − min(0.55, 守.智/360)         （守方智力减伤）
其余 counter / critMul / luck / power(=1) 同物理；可暴击；忽略守方格挡
```

## 五、暴击（由魅力决定）

```
暴击率 = min(0.6, 攻.魅/280) + (处于蓄势 ? 0.35 : 0)
触发 → 伤害 × 1.7
```

猛攻 / 普攻 / 谋攻均可暴击。蓄势的 `×2.0 暴发` 与暴击 `×1.7` 可叠加。

## 六、相克倍率 counter（攻方招式 × 守方「最近姿态」）

> 守方姿态 = 它上一次的攻击招式（猛攻/普攻/谋攻）；用计、格挡后姿态记为「普通」。

| 攻＼守 | 格挡 | 谋攻 | 普攻 | 猛攻 | 蓄力 |
|---|---|---|---|---|---|
| 猛攻 | 0.55 | 1.35 | 1.05 | 1.0 | 1.45 |
| 普攻 | 0.85 | 1.05 | 1.0 | 0.95 | 1.2 |
| 谋攻 | 1.5 | 0.95 | 0.95 | 0.6 | 1.25 |

（实战中守方姿态只会是 普通/猛攻/普攻/谋攻 之一。）

## 七、格挡（必定成功）

```
减伤比例 frac = clamp(0.55 + (己.统 − 敌.武)/250, 0.2, 0.92)
被卸伤害 blocked = round(原伤害 × frac)
实际受伤 = 原伤害 − blocked
己方战意 += blocked（上限 100）   ← 卸掉的伤害全部转化为战意
```

- 仅对**下一次物理受击**生效，用后消失；**谋攻无视格挡**。

## 八、计策（智力系）

**通用成功率** = `clamp(base + (己.智 − 敌.智)/220, 0.12, 0.94)`

| 计策 | 成功率 base | 效果 |
|---|---|---|
| 束缚 | 0.30 | 敌被束缚 `(己.智 − 敌.智 > 45) ? 2 : 1` 回合（暂停出招） |
| 弱化 | 0.45 | 敌攻击 `×(1 − reduce)`，`reduce = min(0.55, 0.22 + 己.智/600)`；时长 `clamp(2 + round((己.智 − 敌.智)/30), 1, 4)` 回合 |
| 疗伤 | 0.62 | 回体力 `round(己.智 × (0.26~0.46 随机))`，不超上限 |
| 蓄力 | 0.66 | 战意 `+round(6 + 己.智 × 0.12)`，并进入「蓄势」（下次出招暴发 ×2.0、暴击率 +0.35） |
| 谋攻 | 0.55 | 见第四节（攻击型） |

- 束缚、弱化为**免费计策**：发动不占行动，同回合两者各可发动一次，之后仍可正常出招。
- 疗伤、蓄力占用本回合行动。

## 九、胜负判定

- 体力 ≤ 0 → KO。
- 自动模拟（阵营大战 / 世界杯 / 快捷观战）出手上限 160；耗尽则比剩余体力，高者胜。

## 十、评级 / 总评（图鉴与界面）

- 单维评级：`SS ≥ 100 · S ≥ 95 · A ≥ 90 · B ≥ 80 · C ≥ 70 · D ≥ 60 · E < 60`
- **总评 = 六维之和**；均值 = 和 ÷ 6（总评徽章按均值取整评级）。

## 十一、友谊系统（自选武将专属）

**获得途径**（主角出战的对应玩法中自动累计，上限 `Bond.MAX_FRIEND = 300`）：

| 互动方式 | 友谊值 |
|---|---|
| 历练 / 2v2 单挑击败对手 | 对手 +5 |
| 2v2 与副将并肩获胜 | 副将 +15 |
| 组队大战同队出战 | 每名队友 +3，获胜再 +3（共 +6） |
| 百人斩击败守将 | 每名被斩守将 +4 |
| 阵营大战并肩存活到终局 | 同阵营幸存同袍各 +2 |
| 世界杯同组交手 | 同组对手各 +3 |
| 国战战至终局仍在麾下 | 同袍各 +3，获胜 +6 |
| 赠送宝物（按「(武将,宝物)」首次赠送计算，不按天） | 按所赠宝物稀有度：普通 +10 / 精良 +30 / 稀有 +60 / 传说 +100 |
| 拜访（每人每【游戏日】限一次，宿营过夜后可再访；未开局时按自然日；无消耗；**友谊达上限 300 后禁用**，按钮置灰并提示「友谊已满」） | 随机 +1~+2 |
| 切磋（武将详情弹窗「⚔️ 切磋」按钮，经典单挑，每人每【游戏日】限一次，耗 1 行动力） | 获胜 +3~+5；落败不加；另见下方「切磋习得」 |

**实际增量与上限显示一致**：`Bond.addF(id,n)` 返回本次实际生效的增量（`min(300, 原值+n) - 原值`），而非传入的名义值 n。切磋获胜、赠送宝物、拜访等一切会展示「友谊 +N」的结算文案均改用该返回值渲染——若对方友谊已在 300 上限附近，文案会如实显示较小的实际增量甚至改为「友谊已至上限」提示，不会再出现友谊早已封顶却仍显示按名义值增长的错位文案。

**友谊等级**：陌生 0 → 相识 30 → 好友 80 → 挚友 150 → 生死之交 300（与招募所需的上限对齐）

**招募**（改为「满上限 + 重金」，不再有中间档可招募或免费档）：
- 友谊须达上限（≥300）才可谈及招募；未满上限时按钮固定置灰显示「🔒 友谊满上限（300）后可招募」
- 招募费 = `武将评分 × 10` 金（无免费档，`Bond.recruitCost`）
- 团队上限 = `5 + ⌊主角等级 / 10⌋`
- 队友用途：2v2 任选副将；同阵营队友在组队大战 / 国战 / 阵营大战必上阵
- **队友不可随意请出团队**；团队已满时再招募新武将，须从现有队友中选一名顶替，才能腾出位置（顶替流程仍走正常的招募友谊门槛与花费）

**切磋习得**：切磋结算时额外判定一次——触发概率随对手当前友谊值线性提升，`1% + (友谊/300)×30%`，友谊 0 时最低 1%、友谊满 300 时最高 31%（`RPG.trySparLearn`）。命中后比较双方按当前实际数值（含装备/惩罚/成长加成，`Armory.geared`）——取败方六维中最高一项，若严格大于胜方同一项，则胜方该项 +1：主角一方走 `RPG.char.alloc` 永久成长（110 封顶保护），对手一方写入战役内 `Campaign.mapState().statGrowth`（与刺杀惩罚 `statPenalty` 同构、符号相反，只影响本局展示与交战、不写回武将图鉴全局数值）。

**赠送宝物细节**（在武将详情页的装备栏为其选择宝物，即为赠送——不再有单独的赠礼弹窗）：
- 只能从玩家自己仓库中「已鉴定且未装备」的宝物里选（避免误赠正被自己或队友佩戴的装备）；宝物本身不再额外消耗金币——它的价值已体现在获取它所付出的代价（掉落运气 / 商店金 / 锻造材料）上。
- 赠出的宝物会直接为对方装备到对应槽位；若对方该槽位已有其他宝物，旧宝物会先卸下退回玩家仓库（不会被覆盖丢失），再换上新赠之物。
- **史实武将的装备槽没有「卸下」选项**，只能用另一件宝物替换（送出去的东西不能白白要回来，只能以新换旧）；「卸下」按钮仅在主角自己的装备栏（角色扮演主页）可用。
- 选择弹窗的「取消」按钮会回到武将详情页而非直接关闭整个信息面板。
- **防刷机制**：友谊奖励按「这件具体的宝物（以其唯一编号识别）是否第一次装到这位武将身上」发放，而非按自然日限次。同一件宝物无论卸下多少次、换给多少位武将、之后又换回来，都只在史上首次装到某位武将身上时给一次友谊——用两件宝物来回横跳无法重复刷分，唯一的加友谊途径是不断获得新的宝物（掉落 / 商店 / 锻造）。

## 十三、宝物系统（自选武将专属）

**五类宝物**：

| 类型 | 加成属性 | 举例 |
|---|---|---|
| 兵器 ⚔️ | 武力 | 青釭剑、方天画戟、村正、正宗 |
| 坐骑 🐎 | 统帅 | 赤兔马、的卢、川中岛骏马 |

| 服饰 👘 | 魅力 | 蜀锦战袍、羽织家纹、凤纹腰带 |

**稀有度与属性加成**（兵器/坐骑/书籍/服饰四类，以及奇珍的「体魄」效果；每件宝物生成时在对应区间内独立随机取值，而非固定单值，`Armory.rollBonus(rarityK)`）：

| 普通 | 55 | +1~+2 | 1000 金 |

| 稀有 | 13 | +6~+8 | 6000 金 |
| 传说 | 4 | +9~+10 | 10000 金 |

**鉴宝**：战场拾获（掉落 / 一统天下 / 世界杯名次奖励）的宝物一律以「❔ 神秘宝物」状态入库，结算提示与仓库列表均不透露名称、稀有度、类型或效果——需在「宝物库·仓库」花 **50 金** 鉴宝，才能查看具体细节，并解锁装备与拆解。商店购买、锻造所得的宝物则即得即鉴定（购买/锻造本身就是明确的选择，无需二次鉴定）。

**宝物库**（角色扮演主页「🏪 宝物库」入口，四个页签，随战役进度变化；与下文全局「宝物阁」不同）：

- **仓库**：浏览已获得的全部宝物（含待鉴定的神秘宝物，固定排在已鉴定宝物之后，不按其尚未揭示的真实稀有度参与排序，避免用出现位置泄露信息）；已鉴定且未装备的宝物卡片底部并列两个各占 50% 宽度的按钮——**拆解**（换取材料：普通1／精良2／稀有3／传说5，二次确认）与**出售**（`Armory.sellItem`，换取金币：市价的 50%，即普通500／精良1500／稀有3000／传说5000 金，二次确认）。出售后该宝物从仓库移除，并以其自身名称/属性重建为一条货摊，加入「商店」页签的 `Armory.data.shop` 货架池，可再被购回（重新购得时属性加成重新随机）。
- **图鉴**：按五类分组展示 100 件模板的发现进度与各自的属性/效果加成区间，未发现的显示「？」。
- **商店**：每日自动刷新 6 件随机货架，明码标价购买（普通1000／精良3000／稀有6000／传说10000 金，对马岛黑市与行脚商队折扣期间八折）；也可花 20 金主动重刷；购买所得直接鉴定。玩家出售的宝物也会补充进这一货架池。
- **锻造**：每类消耗对应材料 6 份 + 40 金锻造一件随机宝物；连续 12 次未出稀有以上，下一次锻造保底出稀有以上；锻造所得直接鉴定。

**装备**：

- **400 名史实武将均可持有并穿戴宝物，不再局限于已招募的队友**：主角在「角色扮演」主页管理装备（可装备/更换/卸下）；任意武将（无论是否入队）在「武将详情」弹窗中、紧接六维数据下方有一排紧凑的单行五槽装备栏，点击即可装备/更换（仅已鉴定的宝物可选，且没有单独的「卸下」选项——只能以新宝物替换，见「友谊系统」）；选择弹窗的「取消」会回到武将详情页而非直接关闭整个信息面板。
- 每件宝物同一时间只能装备给一位武将；属性/效果加成由 `makeFighter` / `makeTroopUnit` 统一按武将 id 查询叠加，实时影响其出战的任何玩法（历练/车轮/百人斩/2v2/阵营/组队/国战/世界杯）——无论该武将是主角、队友，还是随机遇到的对手/守军/敌将。角色扮演主页、武将详情页、**乃至战斗界面点击头像弹出的武将信息**，六维数值旁都会正确标注装备带来的加成（如 `88(+6)`），不受该武将当时是否已处于战斗单位状态影响。
- 武将图鉴列表中，任何已装备宝物的武将都会在友谊值旁显示其携带的宝物图标（不再要求入队）。

## 十四、存档架构与开局向导（武将扮演版第一、二期）

**两层存档**：

| 层级 | 内容 | 存储 key | 生命周期 |
|---|---|---|---|
| 全局层 | 武将图鉴自定义/编辑的武将 | `wujiang_db_v1` | 永不因「新游戏」重置 |
| 全局层 | 宝物阁编辑/自建的宝物模板（overrides + custom） | `wujiang_armory_global_v1` | 永不因「新游戏」重置 |
| 战役层 | 角色（六维基线/加点/等级/经验/成长模式） | `wujiang_rpg_v1` | 随「新游戏」清空 |
| 战役层 | 金币/友谊/队伍/赠礼记录 | `wujiang_bond_v1` | 随「新游戏」清空 |
| 战役层 | 宝物背包（已获得的宝物实例）/材料/商店/保底进度 | `wujiang_armory_v1` | 随「新游戏」清空 |
| 战役层 | 战役元数据（创建时间、开局模式选择） | `wujiang_campaign_v1` | 「新游戏」时重写 |

「首页 → 新游戏」触发 `Campaign.reset(mode)`：清空 `wujiang_rpg_v1`，将 `wujiang_bond_v1`/`wujiang_armory_v1` 重置为初始结构（含金币归零、友谊/队伍清空、宝物背包清空、商店重新刷新），并写入新的战役元数据；若已存在角色会先展示覆盖确认。

**开局三步向导**：

1. **选角色**：
   - 自创武将：随机基线六维（45~80）+ 随机可分配加点（18~30），姓名/阵营自选。
   - 扮演史实武将：从图鉴任选一位，随后二选一成长模式——
     | 模式 | 初始六维 | 成长加成 | 说明 |
     |---|---|---|---|
     | 少年模式 🌱 | 默认值 × 60%（四舍五入，最低 1） | 全部维度成长 ×1；其中默认值最高的两项额外 ×1.5（「本命天赋」，UI 显示 ★） | 加点后数值可突破默认上限 |
     | 巅峰模式 👑 | 默认原值 | 全部维度成长 ×0.5 | 立即可用但后续成长慢 |

   两种途径统一获得 15 点可分配加点（`points`）。成长倍率实现：`allocate(k)` 每次花费 1 点加点时，实际叠加值 = `growthMul × (talents.includes(k) ? 1.5 : 1)`（可能为小数，`eff()` 用四舍五入取整显示/参战）。**六维基础值（不含装备加成）上限 110**——`allocate(k)` 在 `RPG.eff(c,k) >= 110` 时拒绝该维度继续加点并提示「该维度已达上限 110」；切磋习得、武道会名次奖励等一切主角基础值成长途径统一遵循此上限，装备/宝物加成不受影响、仍可叠加超出。
2. **选天下格局**：史实分布 / 群雄乱入，写入战役元数据 `mode.distribution` 并驱动 `buildCityAssignment()` 生成武将城池归属（见下节）。
3. **确认开局**：展示汇总与（若有旧档）覆盖警告，确认后创建角色、生成地图状态，直接进入「天下地图」。

**武将图鉴全局化**：首页「武将图鉴」固定只展示默认六维与资料，不显示友谊/装备等进度数据（`showDetail(g, { global: true })` 会跳过友谊面板、装备栏与装备加成增量计算）。战斗中/角色扮演中弹出的同一详情弹窗不受影响，仍正常显示装备加成与友谊面板。

**宝物阁全局化**：首页「宝物阁」一次性展示全部 100 件模板（不受「是否已发现」限制），可编辑名称/简介/属性（书籍的智力·政治、奇珍的五种效果二选其一）及四档稀有度加成值，也可自建全新模板；所有加成值统一限幅 **1~15**。编辑/自建的结果通过 `Armory.pool(type)`（内置模板 + overrides 覆盖 + custom 自建的合集）实时参与掉落、商店与锻造的随机池，对**下一局及当前进行中的战役**均生效。

**小游戏隔离**：首页「小游戏」的 8 个自由玩法均通过各自模块的 `rpg`/`rpgMode` 标记判断是否处于角色扮演语境；仅当该标记为真时才会调用触碰金币/友谊/宝物/经验的结算方法，因此从「小游戏」入口进入时，无论战役是否存在、进行多少局，都不会读写任何战役存档数据。

## 十五、天下地图与行动力系统（武将扮演版第二期）

**地图结构**：中原二十城（成都/汉中/长安/洛阳/许昌/邺城/徐州/荆州/柴桑/建业/天水/白帝城/上庸/江陵/宛城/汝南/下邳/寿春/合肥/武昌）+ 战国二十城（萨摩/安艺/京都/大坂/尾张/甲斐/骏府/小田原/越后/奥州/肥后/丰后/出云/备前/近江/越前/加贺/美浓/三河/常陆）+ 对马岛（海路中转，连接建业·徐州与萨摩），共 41 个节点，坐标为风格化相对位置（`CITIES` 表），道路为固定邻接关系（`ROADS` 表），非精确测绘；地图背景叠加风格化中日陆地剪影（`CHINA_LAND_PATH`/`JAPAN_LAND_PATH`，同一 0~100 相对坐标系手绘轮廓，非精确测绘但取意接近真实海岸线形状），令城池落于对应陆地色块之上。城池标记由"圆点+悬浮文字"改为矩形徽章（`.map-city`，圆角 3px），姓名直接嵌入徽章内（`.mcity-name` 6px，当前城池 8px），不再另起一行悬浮。地图支持**缩放/拖动**：鼠标拖拽、触控拖拽、双指捏合、滚轮、右上角四个按钮（＋/－缩放、🎯聚焦当前城市——以当前城池坐标为中心放大至 2.2 倍、🗺️显示全景——复位至 1 倍居中）均可调整（缩放范围 1~3 倍，`MapZoom` 模块级状态跨渲染持久），拖拽通过 document 级临时监听实现（不使用 `setPointerCapture`，避免其劫持 click 事件目标导致城池点击失效），拖动后紧邻的一次 click 会被判定为手势尾迹而非有效点击。天数/行动力/金币/名声四项状态统一显示在顶栏（标题「天下游历」与右侧音乐按钮下方另起一行的 `#map-topbar-status` 条，15px，`flex-basis:100%` 强制换行，不论折叠屏展开与否都固定两行，避免和标题/音乐按钮挤在同一行造成堆叠），不再与角色资料卡重复；等级信息移入角色资料卡（姓名下方、评分左侧）；「🏕️ 宿营」按钮不放在顶栏，而是并入当前城池的功能按钮网格（历练/集市/铁匠铺/设施同一排）。地图与城池面板随宽度自适应：< 600px 为竖屏纵向排列（地图 → 角色资料 → 本地武将 → 城池事务，逐段可滚动）；≥ 600px（折叠屏展开/平板/桌面）自动切换为「地图横贯顶部（约占可视高度 30%，吸顶固定）+ 下方左（角色资料+本地武将）右（城池事务）两栏」。全站返回逻辑（`handleBackAction`）现与手机系统/浏览器返回键同步——每次 `showScreen()` 正常推入一条历史记录，硬件返回键触发的 `popstate` 若检测到弹窗打开则仅关闭弹窗（不消耗画面层级），否则复用与左上角返回箭头完全相同的路由规则。

**武将城池分布**（`buildCityAssignment(distribution, seed)`）：

- 每名武将按其阵营被分配到该阵营 20 城之一，永久固定于当局战役（不随时间迁移）。
- **史实分布**：约 180 位主要武将（三国五虎上将/五子良将/江东十二虎臣/主要谋士，及日本主要大名与重臣，如"蜀→成都/汉中""织田→尾张""武田→甲斐""岛津→萨摩"等）依据 `CITY_HINTS` 表指定具体城池，覆盖范围为第二期上线时的原 20 城；第三期后续新增的 20 城（天水/白帝城等 10 座三国要地 + 肥后/丰后等 10 座战国分国）暂未逐一手工考据归属，其上武将全部按姓名字符串哈希稳定映射——同一局内固定，跨局也固定。其余次要/配角武将同样按哈希兜底映射到同阵营城池。这是风味化设计，非严谨考据。
- **群雄乱入**：全部武将（含前述主要武将）按"姓名 + 本局创建时间戳"哈希重新洗牌，每局分布都不同。

**渐进登场**：新游戏开局时，按城池分组，每城随机现身 2~3 名武将（`Campaign.reset` 内实现），其余「未登场」。此后每次宿营，若累计天数跨过登场节点（初始第 6 天，此后每 +5 天一次），会从全部「未登场」武将中随机揭示 3~5 名（不区分城池、不按名声分层——任何一位高手都可能在游戏早期就现身），并以「⚡ 天下快报」提示。城池面板「本地武将」区块以「已现身X/Y」精简展示当前城池数量，条目仅显示完整姓名（不再附带大字单姓头像图标），每行固定 3 个（`.mc-roster.narrow.triple`）；名录下方是与「🎭 角色详情」同一 `.cup-go` 样式的「🌐 全部武将」大按钮，打开独立的 `#screen-allgen` 页面——与武将图鉴（`#screen-db`）同一 `.db-table` 表格风格（可按列排序、三国/战国/全部筛选、姓名搜索），但去除新增/编辑/删除等修改操作，六维与评分改用 `Armory.geared(g, g.id)` 叠加当前已装备宝物后的实时数值（而非图鉴默认值），并新增「友谊」「所在城」两列；点击任意一行弹出的详情为**只读视图**（`showDetail(g,{readonly:true})`）——不显示拜访/切磋/招募等交友板块，携带宝物槽位仅供查看、不可点击更换。

**武将自行迁移**（`MapUI.wanderGenerals`）：每次宿营，每名已现身武将有 3% 概率迁往其阵营内、与当前城池相邻（`adjCities`，不含对马岛海路）的另一城池，使天下版图持续流动，不会永远静止在初始/首次登场的城池。迁移不受名声/行动力影响，纯随机触发，若当日无武将现身通知（天下快报），有武将迁移时会改为提示迁移人数。

**行动力（⚡ Action Point）**：

| 项目 | 数值/规则 |
|---|---|
| 初始上限 | 1 点/天 |
| 消耗 1 点的行动 | 移动一段路、历练（需先解锁，见下）、车轮/百人斩/2v2/阵营/组队/国战/世界杯（均在 `RPG.xxx()` 入口真正开战前扣减，「再来一次」链式点击同样计费）、接取悬赏、设施挑战、切磋（见「友谊系统」） |
| 免费行动 | 拜访本地武将（点击名录进入详情，含赠礼/招募）、逛集市、逛铁匠铺、查看悬赏榜、宿营 |
| 恢复方式 | 「🏕️ 宿营」：天数 +1，行动力回满；行动力耗尽时相关按钮自动置灰提示 |
| 上限成长 | `Campaign.recalcApMax()` = 1 + ⌊名声阶梯 ÷ 2⌋(0~4) + 已装备的行动力奇珍(传国玉玺/九鼎二选一传说品阶，+1)，封顶 6；武道会首冠不再贡献行动力上限；上限提升时同步补给当天差额，即时可用 |

**历练解锁门槛**：历练（`RPG.train()`，随机对手练级）需名声达到「小有名气」阶梯（`RPG.TRAIN_FAME_TIER = 2`）才开放；未解锁时地图「历练」按钮显示 🔒 与所需阶梯提示。开局阶段的名声/经验来源是拜访、切磋、悬赏与设施挑战——历练是这些手段打出声望后解锁的自由练级方式，而非起手可用的无限刷分入口。

**城池通用/特色功能**：📍拜访（本地已现身武将名录，复用既有武将详情弹窗）、🏋️历练、🏪集市（**本地货摊**：每城每游戏日按种子生成 4 件货品（名声达「略有耳闻」第 1 阶起为 5 件），价格按城市行情系数浮动——对马岛黑市固定 0.8，其余城按城名哈希稳定落在 0.90~1.20，行脚商队奇遇期间取更低者；已购摊位当日售罄，宿营翌日全城重新上货）、⚒️铁匠铺（**各城专精一类宝物**：锻专精类只需材料 5+30 金（名声达「名满天下」第 7 阶起再减为 4+20 金），其余类型 6+40 金，保底进度与宝物库锻造共享）、📋悬赏榜（每城 2~3 条，名声达「名动一国」第 5 阶时一次性为每城 +1 条永久空缺，见下节）、🏯特色设施（每城一项，见下节）。

**导航层级与返回逻辑**：天下地图为战役的顶层主界面（「新游戏」「继续游戏」均直接进入）；角色详情（原「角色扮演」主页，含六维加点/团队/装备）为地图下的子页，通过状态条「🎭 详情」按钮进入，其返回按钮固定回到地图；宝物库（仓库/商店/锻造）可从地图（集市/铁匠铺）或角色详情页两处进入，返回按钮通过 `homeBase` 机制智能回到发起它的那一层，而非固定某一处。所有战斗结算弹窗的「返回养成」类按钮统一改为调用 `goHome()`，同样遵循这一"回到发起处"的规则。

## 十七、天下历、城池归属、边境阵营大战与刺杀

**天下历**：`Campaign.mapState().day` 从第 1 天起递增，`calYMD`/`calLabel` 换算为每年 12 月、每月 30 天的「X 年 X 月 X 日」展示（年份自 0 起、月与日自 1 起）；`isMonthEnd(day)` 即 `day % 30 === 0`，为下述边境阵营大战的触发条件。

**城池归属**：`m.cityOwner`（城池 id → `'cn'|'jp'`）与武将分布沿用的静态 `c.side` 是两套独立数据——前者随战事可变，后者恒定不变。`initCityOwner()` 令唯一的海路中转站对马岛（`c.side === "sea"`）初始归属战国一方，其余城池归属与固有阵营一致。

**边境战触发**：每逢月末宿营（`isMonthEnd` 成立），`borderEdges(m)` 找出 `ROADS` 中两端 `cityOwnerSide` 不同的所有边——初始仅对马岛↔建业、对马岛↔徐州两条，随城池易主自然向外蔓延为新的前线；仅当边两端阵营各自都有「已现身武将」时才计入候选。**每月最多只从候选边中随机挑一条开战**（`edges[randInt(0,edges.length-1)]`），不会同时爆发多场。

**边境战结算——不亲征（或与主角阵营无关）**：双方各取「已现身武将」（不限本地驻军）随机等量人马（`Math.min(己方数,敌方数)`），依次用 `js/engine.js` 的 `autoBattle(g1,g2)` 单场极速判定，败者出列直到一方队列耗尽，即时弹出战报。

**边境战结算——亲征**：主角选择率团队亲征时，改用**组队大战**（`TeamBattle`）而非阵营大战——武将选择原则不变（双方均从「已现身武将」中取，主角与同阵营队友必上阵，其余随机补足，双方数量相等），仅因 `TeamBattle` 单场最多 10 将上阵，双方数量额外取 `Math.min(己方,敌方,10)`（与国战攻城 `Conquest` 的做法一致）。调用 `TeamBattle.begin(mine, heroSide, {exact:true, enemies:theirs, rpg:true, onDone})`——`exact:true` 跳过其默认的「不足 10 人从全库补满」逻辑，`rpg:true` 供全局返回键逻辑判断该局应回到发起它的那一层（地图）。战斗打完后经 `onDone(result)` 回调（`result = {playerWon, mySurvivors, theirSurvivors, kills}`）取代默认的「再来一场/返回菜单」收尾；亲征取胜额外获得名声 +20、经验（本方战场斩获数×15 + 60）与金币（30 + 斩获数×5），主角是否阵亡由 `mySurvivors` 中是否仍含 `id===-1` 判定，若阵亡则文案标注「阵中负伤」但不影响整体战果判定。

**夺城后的部署调整**：无论哪种结算方式，胜方都通过共用的 `applyBorderWarOutcome(m, edge, winnerSide)` 处理后续——`m.cityOwner[被占城]` 改写为胜方阵营；败方原驻守该城（`m.assign[gid] === 被占城`）的己方武将，各自退守至与该城相邻、仍由己方持有的城池（`adjCities` + `cityOwnerSide` 过滤，找不到时原地不动）；胜方从「已现身武将」中随机挑选 2~4 名（`randInt(2,4)`，排除已在该城的）进驻该城成为新的本地驻防。

**边境犒赏**：只要 `winnerSide === heroSide`（不亲征的自动模拟结算、亲征的组队大战结算均判定同一条件），主角额外获得 `MapUI.BORDER_WAR_WIN_BONUS = 1000` 金固定犒赏（`Bond.addGold`），与战斗本身的经验/金币结算分开显示（不亲征战报单独一行「🏆 己方大捷，边境犒赏」；亲征战报并入「你的战果」区块）。

**潜入敌境与刺杀**：`showDetail` 按目标与主角阵营关系互斥展示交友区（同阵营）或刺杀区（异阵营，非只读视图），刺杀按钮标注「-1⚡」耗行动力（与切磋同规格，行动力不足时置灰）。刺杀即经典单挑（`m.activeAssassin` 记目标，`Bond.data.assassinDay` 限每人每日一次），结算走 `RPG.onBattleEnd`：取胜则从六维中随机一维给敌将扣 1~3（`randInt(1,3)`）、主角名声 +15，落败则反过来扣主角自己同样一维 1~3；数值写入战役内 `m.statPenalty`（键为武将 id 或 `"hero"`），由 `Armory.geared()` 在既有装备加成之上一并减去，六维下限 10，且只影响本局战役展示与战斗数值、不写回武将图鉴全局默认值；展示侧的装备增量标注（`statRow`/`rpg-dim` 两处 `rd-gear`）按数值正负显示 `+N`/`-N`，避免负值被固定 `+` 前缀拼接成 `+-N`。为避免通用胜利结算里"不打不相识"友谊加成意外拉近与敌将关系，`Bond.addF` 在该分支被限定为仅同阵营对手才执行。

## 十八、阵营大战亲历模式与武将大会

**阵营大战亲历询问**（`War.start(hero, opts)` 内建功能，历史沿革见下）：快捷模式下每逢队列轮到的一方是主角本人（`id===-1`）或现有队友（`Bond.inTeam`）时，`isHeroOrMate()` 判定为真，弹窗 `askJoinDuel()` 询问「亲自应战」（走 `autoPlayBattle` 进入经典单挑画面）或「自动观战」（照常 `autoBattle` 瞬间结算）；`War.abort()` 对该弹窗 pending promise 做清理，避免中途退出留下悬空的 `await`。**注**：角色详情页曾短暂新增过「⚔️ 阵营大战」快捷入口以让这一功能可达，第十二轮已按反馈移除该入口——`War`（阵营大战）此后仅能从首页「小游戏」自由试玩进入，`askJoinDuel` 机制在该入口下同样生效，只是不再有 RPG 角色数据参与。

**武将大会触发**：`isTournamentDay(day)` 即 `calYMD(day)` 的 `month∈{2,5,8,11} && dom===1`，在 `camp()` 中于月末边境战判定之前检查（`MapUI.checkTournament`）。若当前无任何已现身武将则直接跳过（凑不出赛程）。

**报名与费用**：`checkTournament` 弹窗报名费按 `Math.round(ratingScore(heroGeneral())×2)` 计算（与招募费用同一量级公式），金币不足时报名按钮置灰。选择报名则立即 `Bond.spend(fee)`，若最终战报判定「杀入四强」（`heroPlacement().label` 命中 `/夺冠|决赛|半决赛/`，即至少打入四强赛轮次）则全额 `Bond.addGold(fee)` 退还。

**参赛名单**：除主角外的 31 席从 `m.appeared` 已现身武将（不分阵营、含队友）中按 `ratingScore(g)` 从高到低排序取前 31 名（`pool.slice().sort((a,b)=>ratingScore(b)-ratingScore(a))`，第十四轮由原先的随机抽取改为择优选取，使武将大会真正汇聚"当下天下最强的一批人物"），不足则以 `MapUI.byeFighter(i)` 生成的「轮空」占位武将（六维均为 10，几乎必败）补满，保证赛程恒定 32 强；`Tournament.size` 在开赛前强制重置为 32，避免沿用小游戏自由试玩残留的规模设置；`Tournament.draw()` 仍会 `shuffle(this.participants)` 打乱分组种子顺序，但只影响对阵签位，不影响已经选定的这 32 位参赛者本身。

**不参加时的后台判定**：`Tournament.simulate(parts)` 是新增的无 UI 纯逻辑赛程模拟（小组循环赛取前二 → 单败淘汰，全程 `autoBattle`），供主角选择不参加时仍在后台产生冠亚军并发放奖励，不占用/不影响 `Tournament` 模块的交互状态（`this.participants` 等）。

**参加时的完赛回调**：`Tournament` 新增 `onDone` 可选回调（`settlePrediction` 内两处收尾分支均优先判断 `this.onDone`），设置后取代默认的 `RPG.onCupResult`／试玩战报弹窗，交由调用方（`MapUI.runTournament`）自行展示战报与结算奖励。

**冠亚军奖励**：`MapUI.applyTournamentPrize(m, general, statAmt, isChampion)` 统一处理——冠军 `statAmt=3`、亚军 `statAmt=1`。主角六维加成经 `grantHeroStatGrowth`（`RPG.char.alloc[dim] += 加成`，逐维检查 `RPG.eff(c,k)<110` 挑选未封顶的维度，已全部封顶则加成为 0，不会突破上限）直接永久生效；非主角武将经 `grantNpcStatGrowth` 写入战役内新字段 `m.statGrowth[gid][dim]`（与 `m.statPenalty` 同构、符号相反），由 `Armory.geared()` 一并叠加展示，不写回武将图鉴全局数值。冠军额外触发 `grantChampionTreasure`：主角冠军 `Armory.guaranteedItem("legend")` 直接入库（未鉴定）；非主角冠军随机选定宝物类型 `Armory.makeItem(typeK,"legend")`，与其当前该类型已装备宝物比较稀有度/加成（`Armory.RARITIES` 顺序 + `bonus` 数值），更好则直接为其换装（原装备退回未装备状态），持平或更差（如双方皆为传说级）则改发一次 `grantNpcStatGrowth(m, championId, 3)`——与冠军的基础 +3 各自独立发放，因而运气不佳时会叠加为同一维度或不同维度累计 +6。

**宿营反向夜袭**：宿营结算顺序为「天数+1 → 月末边境战判定 → 夜袭判定 → 常规天下快报/迁移提示」。若当前城池本地已现身武将中存在敌方阵营成员，15% 概率弹出夜袭（随机挑一名敌将发起），「应战」直接复用与主动刺杀完全相同的结算通道，胜负判定对称。

**武道会与「天下无双」终局**：世界杯 32 强需名声达「威震一方」方可报名；首次夺冠仍记 `m.cupWon = true`（第十四轮起不再贡献行动力上限，见上「行动力」表格，该标记现仅用于下述终局条件判定）。`RIVAL_NAMES` 八大高手——吕布、关羽、张飞、赵云、织田信长、武田信玄、上杉谦信、本多忠胜（按姓名匹配，兼容武将图鉴改名/重建默认库）——在任意经典单挑/2v2 中击败即记入 `rivalsDefeated`。集齐八大高手战绩 + 至少一次武道会夺冠，触发一次性「🏆 天下无双」终局庆典弹窗（`ending` 标记防止重复触发），关闭后照常继续自由游历，不影响后续任何玩法。

## 十九、城市产业（城市经营一期）

**数据结构**：全存战役层 `m.estate = { [cityId]: { type, lastDay, pending, manager, matCarry?, matPending? } }`，新开局重置；旧档由 `Campaign.ensureMap()` 自动补建空档。每城产业类型由 `CITY_ESTATE`（40 城风味映射，对马岛无产业）固定给出，五类参数见下表（`ESTATE_TYPES`）：

| 类型 | 置办价 | 基础日进 | 特性 |
|---|---|---|---|
| 🍶 酒馆 tavern | 1200 金 | 35 金 | 无 |
| 🐎 马场 ranch | 1600 金 | 48 金 | 无 |
| ⛏️ 矿山 mine | 800 金 | 18 金 | 每满 5 天积攒 1 份本城铁匠铺专精材料（`Armory.TYPES[hashStr(cityId)%5]`，与铁匠铺同式），材料待收上限 3 份 |
| 🌾 农庄 farm | 800 金 | 25 金 | 无 |
| 🚢 商队 caravan | 2000 金 | 60 金 | 收取时实得 = 待收 × (0.6~1.4) 随机行情 |

**核心规则**：
- 置办耗 **1 行动力** + 置办价，敌占之城不可置办（按钮置灰）；收取**不耗行动力**，但必须身在该城。
- 收益**懒结算**（`Estate.accrue`）：任何读到该产业的时机（地图渲染总览、打开面板、收取、易主结算）按 `m.day - lastDay` 补记；待收上限 = 当前日进 × **15 天**，满仓即停。
- **掌柜**：候选 = 已现身 + 友谊满 300 + 不在团队 + 未在他处任职（不限阵营）；加成 = `min(50, round((智力+政治)/4))`%（取 `Armory.geared` 装备后数值），日进 = `round(基础日进 × (1+加成/100) × 繁荣倍率)`。委任/更换/解任前都会先按旧费率结清（`accrue`）。掌柜上任即 `m.assign[gid]=cityId` 常驻本城：`wanderGenerals` 跳过、边境战败军撤离也跳过；被 `Bond.recruit` 招募入队时经 `Estate.onRecruit` 自动卸任。**代收**（第十六轮起）：有掌柜的产业在 `accrue` 时金币不进待收而直接 `Bond.addGold` 入账（商队行情按整段一次判定），累计代收额记入 `est.banked` 供面板展示，不受 15 天积攒上限约束、也不触发经营事件；矿山材料不在代收之列，仍走 `matPending` 由本人收取。
- **查封**：`cityOwnerSide(m,cityId) !== 主角阵营` 即查封——不积攒、不可收取、不可变卖；`applyBorderWarOutcome` 在改写 `m.cityOwner` **之前**先 `Estate.accrue(m, capturedCity)`，保证易主前的天数不漏记、占领期间不多记；夺回后自动恢复积攒（已积待收原封保留）。
- **变卖**：非查封状态随时可卖，回收 = `round(置办价 × 0.5) + 全部待收`，档案删除后可再次置办。
- **总览**：地图左栏角色卡下方「🏪 名下产业 N 处 · 待收 X 金」，渲染时顺带对所有产业做一次懒结算。

## 二十、城市繁荣度、经营事件与一键拜访（城市经营二期）

**繁荣度（`Prosper`，存 `m.prosper[cityId] = {lv, pts}`）**：
- 星级 1~5；未记录的城取初始档：`PROSPER_INIT_HIGH`（洛阳/许昌/成都/建业/京都/大坂/尾张/骏府）3 星，`PROSPER_INIT_LOW`（白帝城/上庸/奥州/肥后/常陆）1 星，其余 2 星。
- 繁荣点来源：设施挑战获胜 +1（duel/duo、车轮全清、阵营/组队/国战的设施加成分支）、完成悬赏 +1（`completeBountyReward`）、置办产业 +2（`Estate.buy`）、己方边境战夺城/光复 +2（`applyBorderWarOutcome`，仅 `winnerSide === 主角阵营`）。每满 5 点升 1 星（可连升），满 5 星封顶后不再累计。
- 影响：产业收益倍率 `×(0.8 + 0.1×lv)`；集市货摊 `≥4 星 +1`（叠加名声第 1 阶的 4→5）；铁匠铺专精工钱 `≥4 星 ×0.7`（叠加名声第 7 阶的 30→20，四舍五入）。城名旁金色 ★ 实时显示。

**经营事件（仅亲自收取触发，掌柜代收静默）**：`MapUI.collectEstate` 单次 `Math.random()` 按固定区间判定——
| 区间 | 事件 | 效果 |
|---|---|---|
| [0, 0.12) | 🗡️ 强人劫掠（农庄免疫，命中视同无事件） | 待收先经 `Estate.seizeForRaid` 清账押走（商队行情此时判定）；**应战**走经典单挑（对手为 `makeRaider` 生成的浪人，六维贴主角 −12~+8 浮动、id=-77 不入图鉴），结算经 `RPG.onBattleEnd` 的 `m.activeEstateRaid` 通道：胜 = 追回 **2 倍** + 名声 +5，败 = 保住 **50%**；**破财消灾** = 保住 **70%**。矿山材料两种结局均全额送回 |
| [0.12, 0.22) | 📈 大市 | 本次金币收益 ×2（`Estate.collect` 的 `goldMult=2`） |
| [0.22, 0.32) | 🎉 贵客临门（酒馆特有） | 随机一位己方已现身武将友谊 +10（按实际增量），收益照常 |
| 其余 | 无事件 | 照常结算 |

**一键拜访（`MapUI.oneClickVisit`）**：本地武将区新增大按钮，一次访遍当前城池所有「已现身 + 己方阵营 + 友谊未满 300 + 今日未访」的武将——逐一写入 `visitDay` 并 `addF(randInt(1,2))`，与逐个拜访完全同规则同额度（无额外收益，纯省点击），汇总一条 toast（超过 8 人截断显示）。不消耗行动力。

## 二十一、城建捐修、产业升级链、守将与敌境自营（城市经营三·四期）

**城建捐修（`Buildings`，存 `m.builds[cityId][type] = lv`）**：
- 除对马岛外每城开放 3 种公共建筑：医馆/书院/驿站/城墙/演武场**五选三**，按 `hashStr(cityId+"|"+type)` 排序稳定选取（同一城每局相同）。
- 修至 1/2/3 级花 **500/1000/2000 金 + 0/1/2 份本城铁匠铺专精材料**，不耗行动力，每级 +2 繁荣点。
- 效果一览（均要求**当前所在城归属己方**，城墙除外——攻守对称）：

| 建筑 | 1 级 | 2 级 | 3 级 | 接入点 |
|---|---|---|---|---|
| 🏥 医馆 | 回魂丹 80 金 | 65 金 | 50 金 | `maybeRevive`（基准 100 金） |
| 📚 书院 | 单挑经验 ×1.1 | ×1.2 | ×1.3 | `onBattleEnd`（历练/切磋/悬赏/设施单挑均计） |
| 🏇 驿站 | 开通驿路 | 驿费 −25% | 驿费 −50% | 两端均须己方且建有驿站；1⚡ + 60 金×BFS 最短跳数 |
| 🏯 城墙 | 守备 +2 | +4 | +6 | 边境战报模拟中本城一方全员六维加成（敌占期间为敌所用；亲征不加成——胜负由你亲手打出） |
| ⚔️ 演武场 | 设施名声 +4 | +8 | +12 | duel/duo、车轮全清、阵营/组队/国战五处设施奖励分支 |

- 建筑归属跟随城池：城破为敌所用（面板只读可查看），夺回后原级保留、即刻恢复。

**产业升级链（`m.estate[cityId].lv`，缺省 1）**：
- 三段名号：酒馆→酒楼→天下名楼 · 马场→名驹牧场→天下第一厩 · 矿山→深矿→天下宝矿 · 农庄→庄园→天下粮仓 · 商队→商行→天下商盟。
- 扩建价 = 置办价 ×1.25（升 2 级）/ ×2（升 3 级），且城市繁荣须达 **★★ / ★★★**；扩建 +1 繁荣点、不耗行动力。
- 收益倍率 ×1 / ×1.6 / ×2.4 乘入 `dailyRate`（与掌柜加成、繁荣倍率连乘）；矿山出材周期 5/4/3 天一份、待收上限 3/4/5。
- **变卖回收 = 累计造价（`cumCost`，含历次扩建）× 0.5 + 全部待收**（原为仅置办价五成）。

**守将（`Guard`，存 `m.guards[cityId]=gid` 与 `m.captives[gid]={cityId,day}`）**：
- 候选：已现身 + 友谊满 300 + **己方阵营** + 不在团队 + 未任掌柜（守将与掌柜候选互斥）。
- 效果：常驻不云游；边境战报模拟必上阵（置于本方阵前）且六维 +3（与城墙叠加）；亲征时若战线上己方城池有守将则必编入队伍。
- **城破被俘**：从天下名录暂时消失（不可拜访/上阵/委任），囚于陷落之城。营救三途：付赎金（**评分 ×5 金**）、劫牢营救（1⚡ 经典单挑 vs 狱吏[主角六维 −8~+10 浮动]——胜 = 放还 + 名声 +10 + 友谊 +20，败 = 留待再来）、己方夺回该城自动放人。入队自动卸任。

**敌境自营（`AIDev.tick`，每次宿营执行）**：
- 敌方治下城市随机抽 **2 座**，各 **60%** 概率发生一次发展动作：45% 繁荣 +1 点（静默，升星才播报）/ 30% 修筑升级城建 / 25% 兴办·扩建敌营产业（`m.npcEstate[cityId]={lv}`，上限 3 级，与玩家产业互斥占位）。期望约 1.2 个动作/宿营，动向以「🏗️ 敌境风闻」并入宿营提示。
- **夺城红利**：城建直接为你所用；敌营产业可按**累计造价 × 0.5** 接管为自家产业（等级保留，耗 1⚡、+2 繁荣点）；未接管期间闲置，敌方夺回则复归敌营。

## 二十二、边境战亲历判定、医馆扩效、宿敌系统与杂项（第二十轮）

**边境战（重写 `resolveBorderWar`）**：
- 不再弹「是否亲征」二选一，改由**主角所在城池自动判定**：`m.curCity` 恰为交战两城之一且该城归属己方 → 亲历（`heroForced`）——主角+同阵营队友由 `War.start` 的既有强制上阵逻辑排在本方阵前；否则主角以 `heroGeneral()` 战力**混入本方候选池随机抽取**（不带队伍、无强制）。
- 无论亲历与否，战斗一律走**阵营大战（War）小游戏快捷模式可视化演出**（`customRoster` 双方等量、守将置前 +3、城墙 ±2×lv 全员加成——主角自身[id=-1]不叠加城墙/守将附加，其数值由 `heroGeneral()` 独立精算）；轮到主角/队友出战时仍会弹「亲自应战/自动观战」。
- 战报：亲历 = 个人战果（斩获×15 经验、胜 +20 名声、胜 30+杀×5 金）+ 边境犒赏；未亲历 = 简报 + 己方胜时照发犒赏 ≥1000 金。`War.rpg` 战后强制置真，保证硬件返回键回天下地图。

**医馆扩效（在原「回魂丹折价」上叠加，均要求所在城归属己方）**：
| 等级 | 回魂丹 | 单挑体力回复 | 带兵兵力恢复 |
|---|---|---|---|
| 1 | 80 金 | +2/回合 | 安抚/驰援 +3% 上限 |
| 2 | 65 金 | +4/回合 | +6% |
| 3 | 50 金 | +6/回合 | +9% |
- 体力回复经 `RPG.heroGeneral()` 注入 `regenBonus`（与奇珍「气血回复」同通道叠加，engine.js `endTurn` 结算）；兵力恢复在 `TeamBattle.doScheme` 中对己方 rally/reinforce 命中后追加（仅角色扮演模式）。

**地图**：新增道路 `["tsushima","higo"]`——对马岛两侧各两条航线（建业/徐州 ↔ 对马 ↔ 萨摩/肥后），渡海风暴规则照旧覆盖全部四条。

**城池功能区**：`.map-menu-free`/`.map-menu-ap` 由 3 列改 **2 列**；「历练」按钮移除（`RPG.train` 保留但无地图入口，练级走切磋/悬赏/设施/边境战等）。

**全部城市**：新增「日进」列（`dailyGold` 可排序）——已置办产业按当前费率（含等级/掌柜/繁荣倍率）、敌营产业按 `npcDailyRate` 标注（敌）、未置办显示基础费率为潜力值、查封显示「查封」。

**宿敌（`Nemesis`，存 `m.nemesis = {id, wins, nemesisWins, ambush, finaleDone}`）**：
- **结缘**：`RPG.onBattleEnd` 中主角首败于 `side !== 主角` 且 `id ≥ 0` 的武将即自动指定（已有宿敌不换）；兜底——名声达第 3 阶「声名初显」后每次宿营 8% 概率从敌方已现身武将中随缘指定。
- **战力**：`buffedOpponent` = 真实数值（含装备）+ 全维 `round((主角等级-1)×1.5) + 已被击败次数×4`，仅本场生效。
- **寻衅**（每次宿营 15%，终局已了则停）：⅓ 扬言拦路（置 `ambush` 标记，下次 `moveTo` 时强制应战，行动力照扣、照常抵达）、⅓ 抢先夺走本城随机一条悬赏（该槽位换新）、⅓ 踏营下战书（宿营流程末弹应战确认）。
- **对决结算**（`m.activeNemesis` 通道）：胜 = wins+1、+20 名声、80+wins×20 金；败 = nemesisWins+1。**wins 满 3 触发「恩怨了结」**：+100 名声（20+80）、必得一件传说级宝物（`guaranteedItem("legend")`，未鉴定入库）、`finaleDone=true` 后不再寻衅（宿敌卡仍可主动重温旧怨）。
- **UI**：地图左栏角色卡下方宿敌卡（头像/战绩/进度 + 「🗡️ 下战书」按钮，耗 1⚡）。

## 二十三、边境战组队化、宿营夜报、宿敌降幅与弹窗防误触（第二十一轮）

**边境战改用组队大战（TeamBattle）**（再次重写 `resolveBorderWar`，替代第二十轮的 War 快捷模式方案）：
- 参战名单构建不变（已现身武将、守将置前 +3、城墙 ±2×lv、主角亲历判定同前），但**每方上限收紧为 10 将**（`count = min(双方池, 10)`，组队大战棋盘规模），经 `TeamBattle.begin(myRoster, heroSide, { exact, enemies, rpg, observe, onDone })` 开打。
- **主角上阵（亲历强制 or 未亲历被抽中）→ 可亲自指挥的组队大战**：带兵攻击/计谋/挑唆/征兵全套操作可用，也可随时「委托 AI」；**主角未上阵 → `observe` 观战模式**（`begin` 新增 opts.observe → 开局即 `delegated=true`，己方全程 AI 代打，AI 间挑唆单挑亦以旁观模式自动演完）。
- 战报：主角上阵 = 个人战果（己方总歼敌 `kills`×12 经验、胜 +20 名声、胜 30+kills×4 金 / 败 10 金）+ 胜方犒赏；未上阵 = 简报 + 己方胜时照发犒赏。夺城/撤军/进驻/守将被俘等结算逻辑（`applyBorderWarOutcome`）不变。
- 返回键归属：`opts.rpg=true` 由 TeamBattle 自身语义承担，第二十轮 `War.rpg` 强制置真的补丁随 War 路径一并退役（War 的 `customRoster` 能力保留为通用接口）。

**宿营夜报**：宿营产生附加消息（敌境风闻/宿敌风闻）时，由一闪而过的 toast 改为「🏕️ 宿营夜报」弹窗逐条列出，点击屏幕任意处（或「知道了」）关闭；无附加消息时仍走原 toast。

## 三十七、野战演武 UI 精修 + 将魂冷却随将浮动 + 名将再倍增至 40 位（第三十五轮）

按用户五点反馈实施：

**① 技能钮移至战线框线左右两侧、加大并写出技能名**：原先技能钮位于姓名行下方单独一行（`.fb-lane-skl`，占满一行高度），改为 `.fb-lane` 整体变为横向 flex 容器——`skillBadgeHtml("my",k)` / 战线本体 `.fb-lane-body` / `skillBadgeHtml("foe",k)` 三者并排，技能钮尺寸由 24×24 的圆形小图标放大为 50×52 的圆角方形按钮，内部 `.face` 承载图标（`.ic`）与技能名（`.nm`，两行内自动换行），冷却环仍用 `conic-gradient(--cdpct)` 呈现，套在按钮外层作为 padding 边框。此举同时省下原先技能行占用的整行高度，为 ③ 的战线内部精简腾出空间。

**② 将魂冷却按持有者统帅（`tong`）浮动**：新增 `FieldBattle.warCD(g) = clamp(5, 12, round(13 - tong/10))`——统帅约 40 时冷却 9 刻，统帅约 100 时冷却 3 刻（clamp 后落在 5~12 刻区间），取代此前全员统一的 `SKILL_CD = 8` 刻硬编码常量（该常量降级为找不到冷却值时的兜底默认）。`tickSkillCooldowns()` 在新任持有者上任、以及每次发动后重置冷却时，都改为调用 `warCD(持有者)` 而非固定值，使不同武将不再机械地同刻步调发动，观感更自然、也让「谁来扛这条线的将魂位」多一层考量（不只看技能强弱，还要看统帅高低决定手速）。

**③ 战线内部布局重排，进一步压缩高度**：框内顺序由「（士气+战线名合一行）→ 兵力条 → 姓名 → 技能」调整为「战线名（独立一行居中）→ 士气（夹在战线名与姓名之间）→ 姓名（移到兵力色条上方）→ 兵力色条」。连同 ① 腾出的技能行高度，单条战线整体更紧凑，五条战线在手机屏幕上占用的总高度明显缩短。

**④ 点将出阵弹窗放大，10 人（5 行）无需滚动**：`pickDuelist()` 的弹窗外层加宽至 `min(520px,94vw)`（原受 `.detail-card` 默认 `min(420px,92vw)` 限制，武将卡片内容常被迫换行拉高单行高度）、外层 `max-height` 提至 96vh，内层武将列表 `max-height` 由 46vh 提至 74vh——两处一起放大后，五行（2 列 ×5 行）可完整显示，不再需要内部滚动条。

## 三十八、将魂冷却差距再拉大 + 战线单行合并布局 + 斗将按钮对换等宽（第三十六轮）

按用户三点反馈实施：①上一轮 `warCD` 5~12 刻的差距仍偏窄（常见统帅 70~90 区间只落在 5~6 刻）；②战线内部结构希望进一步整合——最左/最右为将魂技能钮，中间只留两行：上行「己方姓名·己方士气·战线名·敌方士气·敌方姓名」一字排开，下行兵力色条；③阵前斗将页的「出将应战」「避战不出」两按钮要互换左右位置且等宽。

**① 冷却差距拉大**：`warCD(g)` 由 `clamp(5, 12, round(13 - tong/10))` 改为 `clamp(4, 14, round(14 - (tong-40)/7))`——每 +7 点统帅减 1 刻冷却，全区间覆盖 4~14 刻（原区间实际有效跨度只有 7~8 刻附近的一两刻差距，因为多数武将统帅集中在 70~90 区间、落进旧公式的窄段）。新公式下同一区间（统帅 70~90）对应冷却 6~10 刻，肉眼可辨的差距明显拉开，武将不再扎堆同刻发动。

**② 战线内容合并为两行**：`laneHtml` 由「战线名独立一行 → 士气行 → 姓名行 → 兵力色条」四行压缩为「一字排开行（`.fb-lane-row1`：己方姓名 `flex:1` 省略号截断 + 己方士气 `flex:none` + 战线名 `flex:none` + 敌方士气 + 敌方姓名 `flex:1`）→ 兵力色条」两行，技能钮维持在 `.fb-lane` 最左/最右两端不变。姓名列表较长时以 CSS `text-overflow:ellipsis` 省略，不强行换行撑高整行。

**③ 斗将按钮对换 + 等宽**：`renderDuel()` 中 `#fb-decline`（避战不出）移到 `#fb-fight`（出将应战）之前；`.fb-btnrow .cup-go` 由 `flex:1 1 auto; min-width:100px`（按各自文案内容取自然宽度，导致长文案的「避战不出」比短文案的「出将应战」更宽）改为 `flex:1 1 0; min-width:0`（flex-basis 归零后按相同 grow 比例均分剩余空间，与文案长短无关），两按钮像素级等宽。

**验证**：test24 扩至 54 项全过——新增/更新断言：抓取实际渲染的技能钮标题冷却值计算极差（`Math.max-Math.min`），要求同局内出现 ≥1 刻差距（本轮实测样本极差 3~4 刻，相比上轮观测到的极差通常为 0~1 刻有明显提升）；战线一字排开行的 5 个子元素 class 顺序断言（`fb-lane-my → fb-lmor my → fb-lane-lbl → fb-lmor foe → fb-lane-foe`）；战线框体只剩两个子行（`fb-lane-row1` + `fb-push`）；斗将按钮顺序与像素宽度断言。过程中两次假阳性排查：一次是按钮宽度断言容差过严（`<2px` 卡在恰好 2px 的 flex 分配舍入误差上，实际两按钮已等宽，调宽容差到 `≤3px`）；一次是既有的「雁行阵乱箭蚀敌」断言（无关本轮改动，测的是敌方雁行阵五线平均士气应随刻流失）在观测窗口恰好被前一步「擂鼓」测试撞车（该测试固定对前锋线 +13 士气，若雁行乱箭当刻净流失不足以抵消这波单线大涨，5 线平均值短暂转为上升）——重跑一次即复现为通过，判定为无关本轮改动的既有时序巧合，非回归。回归 test21 22 项、test22 14 项、test23 10 项全过，本轮同样未改动共用引擎签名，改动全部集中在 `js/app.js` 的 `FieldBattle` 模块与 `css/style.css`。

**验证**：test24 扩至 49 项全过——新增「将魂名将专属技能扩充（28 位）」「主动型将魂在战线旁化作技能钮（无「将魂发动」弹窗）」「主动型将魂冷却完毕自动发动（战报含 🌟 将魂发动记录）」三项断言；回归 test21 22 项、test22、test23 全过，确认 `computeDamage`/`endTurn`/`resolveTurn` 的签名调整未影响车轮/百人斩/世界杯/阵营/组队/边境战等共用引擎的其余玩法。
