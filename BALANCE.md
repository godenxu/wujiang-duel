# 战斗数值与公式说明（BALANCE）

> 本文档汇总单挑（轮换出招回合制）的全部数值与公式，便于查阅与调整。
> 变量缩写：体=体力 · 武=武力 · 统=统帅 · 智=智力 · 政=政治 · 魅=魅力；
> 「己」=出招方，「敌/守」=承受方。实现见 `js/engine.js`，评级见 `js/app.js`。

## 一、体力 / 战意

- **体力上限 = 初始体力 = `体`**，归零即 KO。
- **战意初始** = `min(100, round(55 + 政 × 0.45))`
- **每回合末回战意** = `2 + 政 / 22`
- **出招战意消耗** = `max(2, round(招式基础耗 × (1 − min(0.34, 政/300))))`；基础耗为 0 则不耗。

| 招式 | 猛攻 | 普攻 | 格挡 | 谋攻 | 束缚 | 弱化 | 疗伤 | 蓄力 |
|---|---|---|---|---|---|---|---|---|
| 基础耗 | 20 | 0 | 0 | 7 | 12 | 10 | 11 | 4* |

\* 蓄力实际是「恢复战意」，并不扣除。

## 二、先手 / 回合

- **先手**：`spd = 统 + 随机(0~20)`，高者先手；之后双方**逐回合交替出招**。
- 被束缚的一方轮到自己时**跳过该回合**（消耗一层束缚）。

## 三、物理攻击伤害（猛攻 / 普攻）

```
base       = 武                              （只看攻方武力）
mitigation = (1 − min(0.50, 守.统/380))      （守方统帅减伤）
           × (1 − min(0.12, 守.政/1000))     （守方政治微韧性）
counter    = 相克表[招式][守方姿态]
critMul    = 蓄势命中 ? 2.0 : 1.0            （蓄力「暴发」翻倍）
luck       = 随机 0.82 ~ 1.18
atkMul     = 被「弱化」时 <1，否则 1
power      = 猛攻 1.0 / 普攻 0.5

伤害 = base × 0.32 × counter × mitigation × critMul × luck × atkMul × power
```

随后叠加**暴击**（见第五节），最后 `dmg = max(1, round(dmg))`。

## 四、谋攻伤害（智力系，无视格挡）

```
成功率 = clamp(0.55 + (己.智 − 守.智)/220, 0.12, 0.94)
  失败 → 未命中（0 伤害，"被识破"）
  成功 ↓
base       = 己.智
mitigation = 1 − min(0.55, 守.智/360)         （守方智力减伤）
其余 counter / critMul / luck / power(=1) 同物理；可暴击；忽略守方格挡
```

## 五、暴击（由魅力决定）

```
暴击率 = min(0.6, 攻.魅/280) + (处于蓄势 ? 0.35 : 0)
触发 → 伤害 × 1.7
```

猛攻 / 普攻 / 谋攻均可暴击。蓄势的 `×2.0 暴发` 与暴击 `×1.7` 可叠加。

## 六、相克倍率 counter（攻方招式 × 守方「最近姿态」）

> 守方姿态 = 它上一次的攻击招式（猛攻/普攻/谋攻）；用计、格挡后姿态记为「普通」。

| 攻＼守 | 格挡 | 谋攻 | 普攻 | 猛攻 | 蓄力 |
|---|---|---|---|---|---|
| 猛攻 | 0.55 | 1.35 | 1.05 | 1.0 | 1.45 |
| 普攻 | 0.85 | 1.05 | 1.0 | 0.95 | 1.2 |
| 谋攻 | 1.5 | 0.95 | 0.95 | 0.6 | 1.25 |

（实战中守方姿态只会是 普通/猛攻/普攻/谋攻 之一。）

## 七、格挡（必定成功）

```
减伤比例 frac = clamp(0.55 + (己.统 − 敌.武)/250, 0.2, 0.92)
被卸伤害 blocked = round(原伤害 × frac)
实际受伤 = 原伤害 − blocked
己方战意 += blocked（上限 100）   ← 卸掉的伤害全部转化为战意
```

- 仅对**下一次物理受击**生效，用后消失；**谋攻无视格挡**。

## 八、计策（智力系）

**通用成功率** = `clamp(base + (己.智 − 敌.智)/220, 0.12, 0.94)`

| 计策 | 成功率 base | 效果 |
|---|---|---|
| 束缚 | 0.30 | 敌被束缚 `(己.智 − 敌.智 > 45) ? 2 : 1` 回合（暂停出招） |
| 弱化 | 0.45 | 敌攻击 `×(1 − reduce)`，`reduce = min(0.55, 0.22 + 己.智/600)`；时长 `clamp(2 + round((己.智 − 敌.智)/30), 1, 4)` 回合 |
| 疗伤 | 0.62 | 回体力 `round(己.智 × (0.26~0.46 随机))`，不超上限 |
| 蓄力 | 0.66 | 战意 `+round(6 + 己.智 × 0.12)`，并进入「蓄势」（下次出招暴发 ×2.0、暴击率 +0.35） |
| 谋攻 | 0.55 | 见第四节（攻击型） |

- 束缚、弱化为**免费计策**：发动不占行动，同回合两者各可发动一次，之后仍可正常出招。
- 疗伤、蓄力占用本回合行动。

## 九、胜负判定

- 体力 ≤ 0 → KO。
- 自动模拟（阵营大战 / 世界杯 / 快捷观战）出手上限 160；耗尽则比剩余体力，高者胜。

## 十、评级 / 总评（图鉴与界面）

- 单维评级：`SS ≥ 100 · S ≥ 95 · A ≥ 90 · B ≥ 80 · C ≥ 70 · D ≥ 60 · E < 60`
- **总评 = 六维之和**；均值 = 和 ÷ 6（总评徽章按均值取整评级）。

## 十一、友谊系统（自选武将专属）

**获得途径**（主角出战的对应玩法中自动累计，上限 999）：

| 互动方式 | 友谊值 |
|---|---|
| 历练 / 2v2 单挑击败对手 | 对手 +5 |
| 2v2 与副将并肩获胜 | 副将 +15 |
| 组队大战同队出战 | 每名队友 +3，获胜再 +3（共 +6） |
| 百人斩击败守将 | 每名被斩守将 +4 |
| 阵营大战并肩存活到终局 | 同阵营幸存同袍各 +2 |
| 世界杯同组交手 | 同组对手各 +3 |
| 国战战至终局仍在麾下 | 同袍各 +3，获胜 +6 |
| 赠送宝物（按「(武将,宝物)」首次赠送计算，不按天） | 按所赠宝物稀有度：普通 +10 / 精良 +30 / 稀有 +60 / 传说 +100 |
| 拜访（每人每【游戏日】限一次，宿营过夜后可再访；未开局时按自然日；无消耗） | 随机 +1~+2 |
| 切磋（武将详情弹窗「⚔️ 切磋」按钮，经典单挑，每人每【游戏日】限一次，耗 1 行动力） | 获胜 +3~+5；落败不加 |

**友谊等级**：陌生 0 → 相识 30 → 好友 80 → 挚友 150 → 生死之交 250

**招募**：
- 挚友（≥150）：可花金招募，招募费 = `武将评分 × 2` 金
- 生死之交（≥250）：免费入队
- 团队上限 = `5 + ⌊主角等级 / 10⌋`
- 队友用途：2v2 任选副将；同阵营队友在组队大战 / 国战 / 阵营大战必上阵
- **队友不可随意请出团队**；团队已满时再招募新武将，须从现有队友中选一名顶替，才能腾出位置（顶替流程仍走正常的招募友谊门槛与花费）

**赠送宝物细节**（在武将详情页的装备栏为其选择宝物，即为赠送——不再有单独的赠礼弹窗）：
- 只能从玩家自己仓库中「已鉴定且未装备」的宝物里选（避免误赠正被自己或队友佩戴的装备）；宝物本身不再额外消耗金币——它的价值已体现在获取它所付出的代价（掉落运气 / 商店金 / 锻造材料）上。
- 赠出的宝物会直接为对方装备到对应槽位；若对方该槽位已有其他宝物，旧宝物会先卸下退回玩家仓库（不会被覆盖丢失），再换上新赠之物。
- **史实武将的装备槽没有「卸下」选项**，只能用另一件宝物替换（送出去的东西不能白白要回来，只能以新换旧）；「卸下」按钮仅在主角自己的装备栏（角色扮演主页）可用。
- 选择弹窗的「取消」按钮会回到武将详情页而非直接关闭整个信息面板。
- **防刷机制**：友谊奖励按「这件具体的宝物（以其唯一编号识别）是否第一次装到这位武将身上」发放，而非按自然日限次。同一件宝物无论卸下多少次、换给多少位武将、之后又换回来，都只在史上首次装到某位武将身上时给一次友谊——用两件宝物来回横跳无法重复刷分，唯一的加友谊途径是不断获得新的宝物（掉落 / 商店 / 锻造）。

**鉴宝**：战场拾获（掉落 / 一统天下 / 世界杯名次奖励）的宝物一律以「❔ 神秘宝物」状态入库，结算提示与仓库列表均不透露名称、稀有度、类型或效果——需在「宝物库·仓库」花 **50 金** 鉴宝，才能查看具体细节，并解锁装备与拆解。商店购买、锻造所得的宝物则即得即鉴定（购买/锻造本身就是明确的选择，无需二次鉴定）。

**宝物库**（角色扮演主页「🏪 宝物库」入口，四个页签，随战役进度变化；与下文全局「宝物阁」不同）：

- **仓库**：浏览已获得的全部宝物（含待鉴定的神秘宝物，固定排在已鉴定宝物之后，不按其尚未揭示的真实稀有度参与排序，避免用出现位置泄露信息）；已鉴定且未装备的可拆解为材料（普通1／精良2／稀有3／传说5），拆解前需二次确认。
- **图鉴**：按五类分组展示 100 件模板的发现进度与各自的属性/效果加成区间，未发现的显示「？」。
- **商店**：每日自动刷新 6 件随机货架，明码标价购买（普通220／精良520／稀有1100／传说2600 金）；也可花 20 金主动重刷；购买所得直接鉴定。
- **锻造**：每类消耗对应材料 6 份 + 40 金锻造一件随机宝物；连续 12 次未出稀有以上，下一次锻造保底出稀有以上；锻造所得直接鉴定。

**装备**：

- **400 名史实武将均可持有并穿戴宝物，不再局限于已招募的队友**：主角在「角色扮演」主页管理装备（可装备/更换/卸下）；任意武将（无论是否入队）在「武将详情」弹窗中、紧接六维数据下方有一排紧凑的单行五槽装备栏，点击即可装备/更换（仅已鉴定的宝物可选，且没有单独的「卸下」选项——只能以新宝物替换，见「友谊系统」）；选择弹窗的「取消」会回到武将详情页而非直接关闭整个信息面板。
- 每件宝物同一时间只能装备给一位武将；属性/效果加成由 `makeFighter` / `makeTroopUnit` 统一按武将 id 查询叠加，实时影响其出战的任何玩法（历练/车轮/百人斩/2v2/阵营/组队/国战/世界杯）——无论该武将是主角、队友，还是随机遇到的对手/守军/敌将。角色扮演主页、武将详情页、**乃至战斗界面点击头像弹出的武将信息**，六维数值旁都会正确标注装备带来的加成（如 `88(+6)`），不受该武将当时是否已处于战斗单位状态影响。
- 武将图鉴列表中，任何已装备宝物的武将都会在友谊值旁显示其携带的宝物图标（不再要求入队）。

| 层级 | 内容 | 存储 key | 生命周期 |

**开局三步向导**：

1. **选角色**：
   - 自创武将：随机基线六维（45~80）+ 随机可分配加点（18~30），姓名/阵营自选。
   - 扮演史实武将：从图鉴任选一位，随后二选一成长模式——
     | 模式 | 初始六维 | 成长加成 | 说明 |
     |---|---|---|---|
     | 少年模式 🌱 | 默认值 × 60%（四舍五入，最低 1） | 全部维度成长 ×1；其中默认值最高的两项额外 ×1.5（「本命天赋」，UI 显示 ★） | 加点后数值可突破默认上限 |
     | 巅峰模式 👑 | 默认原值 | 全部维度成长 ×0.5 | 立即可用但后续成长慢 |

   两种途径统一获得 15 点可分配加点（`points`）。成长倍率实现：`allocate(k)` 每次花费 1 点加点时，实际叠加值 = `growthMul × (talents.includes(k) ? 1.5 : 1)`（可能为小数，`eff()` 用四舍五入取整显示/参战）。
2. **选天下格局**：史实分布 / 群雄乱入，写入战役元数据 `mode.distribution` 并驱动 `buildCityAssignment()` 生成武将城池归属（见下节）。
3. **确认开局**：展示汇总与（若有旧档）覆盖警告，确认后创建角色、生成地图状态，直接进入「天下地图」。

**武将图鉴全局化**：首页「武将图鉴」固定只展示默认六维与资料，不显示友谊/装备等进度数据（`showDetail(g, { global: true })` 会跳过友谊面板、装备栏与装备加成增量计算）。战斗中/角色扮演中弹出的同一详情弹窗不受影响，仍正常显示装备加成与友谊面板。

**宝物阁全局化**：首页「宝物阁」一次性展示全部 100 件模板（不受「是否已发现」限制），可编辑名称/简介/属性（书籍的智力·政治、奇珍的五种效果二选其一）及四档稀有度加成值，也可自建全新模板；所有加成值统一限幅 **1~15**。编辑/自建的结果通过 `Armory.pool(type)`（内置模板 + overrides 覆盖 + custom 自建的合集）实时参与掉落、商店与锻造的随机池，对**下一局及当前进行中的战役**均生效。

**小游戏隔离**：首页「小游戏」的 8 个自由玩法均通过各自模块的 `rpg`/`rpgMode` 标记判断是否处于角色扮演语境；仅当该标记为真时才会调用触碰金币/友谊/宝物/经验的结算方法，因此从「小游戏」入口进入时，无论战役是否存在、进行多少局，都不会读写任何战役存档数据。

## 十五、天下地图与行动力系统（武将扮演版第二期）

**地图结构**：中原二十城（成都/汉中/长安/洛阳/许昌/邺城/徐州/荆州/柴桑/建业/天水/白帝城/上庸/江陵/宛城/汝南/下邳/寿春/合肥/武昌）+ 战国二十城（萨摩/安艺/京都/大坂/尾张/甲斐/骏府/小田原/越后/奥州/肥后/丰后/出云/备前/近江/越前/加贺/美浓/三河/常陆）+ 对马岛（海路中转，连接建业·徐州与萨摩），共 41 个节点，坐标为风格化相对位置（`CITIES` 表），道路为固定邻接关系（`ROADS` 表），非精确测绘；地图背景叠加风格化中日陆地剪影（`CHINA_LAND_PATH`/`JAPAN_LAND_PATH`，同一 0~100 相对坐标系手绘轮廓，非精确测绘但取意接近真实海岸线形状），令城池落于对应陆地色块之上。地图支持**缩放/拖动**：鼠标拖拽、触控拖拽、双指捏合、滚轮、右上角 ＋/－ 按钮均可调整（缩放范围 1~3 倍，`MapZoom` 模块级状态跨渲染持久），拖拽通过 document 级临时监听实现（不使用 `setPointerCapture`，避免其劫持 click 事件目标导致城池点击失效），拖动后紧邻的一次 click 会被判定为手势尾迹而非有效点击。天数/行动力/金币/名声/等级五项状态统一显示在顶栏（标题「天下游历」与右侧音乐按钮之间的 `#map-topbar-status` 条，含宿营快捷按钮），不再与角色资料卡重复；角色资料卡仅保留头像/姓名/评分，避免同一信息两处展示。地图与城池面板随宽度自适应：< 600px 为竖屏纵向排列（地图 → 角色资料 → 本地武将 → 城池事务，逐段可滚动）；≥ 600px（折叠屏展开/平板/桌面）自动切换为「地图横贯顶部（约占可视高度 30%，吸顶固定）+ 下方左（角色资料+本地武将）右（城池事务）两栏」。

**武将城池分布**（`buildCityAssignment(distribution, seed)`）：

- 每名武将按其阵营被分配到该阵营 20 城之一，永久固定于当局战役（不随时间迁移）。
- **史实分布**：约 180 位主要武将（三国五虎上将/五子良将/江东十二虎臣/主要谋士，及日本主要大名与重臣，如"蜀→成都/汉中""织田→尾张""武田→甲斐""岛津→萨摩"等）依据 `CITY_HINTS` 表指定具体城池，覆盖范围为第二期上线时的原 20 城；第三期后续新增的 20 城（天水/白帝城等 10 座三国要地 + 肥后/丰后等 10 座战国分国）暂未逐一手工考据归属，其上武将全部按姓名字符串哈希稳定映射——同一局内固定，跨局也固定。其余次要/配角武将同样按哈希兜底映射到同阵营城池。这是风味化设计，非严谨考据。
- **群雄乱入**：全部武将（含前述主要武将）按"姓名 + 本局创建时间戳"哈希重新洗牌，每局分布都不同。

**渐进登场**：新游戏开局时，按城池分组，每城随机现身 2~3 名武将（`Campaign.reset` 内实现），其余「未登场」。此后每次宿营，若累计天数跨过登场节点（初始第 6 天，此后每 +5 天一次），会从全部「未登场」武将中随机揭示 3~5 名（不区分城池、不按名声分层——任何一位高手都可能在游戏早期就现身），并以「⚡ 天下快报」提示。城池面板「本地武将」区块以「已现身X/Y」精简展示当前城池数量，右侧「全部武将」按钮打开跨城完整清单（含每人当前所在城池与友谊值，按友谊降序）。

**武将自行迁移**（`MapUI.wanderGenerals`）：每次宿营，每名已现身武将有 3% 概率迁往其阵营内、与当前城池相邻（`adjCities`，不含对马岛海路）的另一城池，使天下版图持续流动，不会永远静止在初始/首次登场的城池。迁移不受名声/行动力影响，纯随机触发，若当日无武将现身通知（天下快报），有武将迁移时会改为提示迁移人数。

**行动力（⚡ Action Point）**：

| 项目 | 数值/规则 |
|---|---|
| 初始上限 | 1 点/天 |
| 消耗 1 点的行动 | 移动一段路、历练（需先解锁，见下）、车轮/百人斩/2v2/阵营/组队/国战/世界杯（均在 `RPG.xxx()` 入口真正开战前扣减，「再来一次」链式点击同样计费）、接取悬赏、设施挑战、切磋（见「友谊系统」） |
| 免费行动 | 拜访本地武将（点击名录进入详情，含赠礼/招募）、逛集市、逛铁匠铺、查看悬赏榜、宿营 |
| 恢复方式 | 「🏕️ 宿营」：天数 +1，行动力回满；行动力耗尽时相关按钮自动置灰提示 |
| 上限成长 | `Campaign.recalcApMax()` = 1 + ⌊名声阶梯 ÷ 2⌋(0~4) + 武道会首冠(+1) + 已装备唯一奇珍数(each+1)，封顶 6；上限提升时同步补给当天差额，即时可用 |

**历练解锁门槛**：历练（`RPG.train()`，随机对手练级）需名声达到「小有名气」阶梯（`RPG.TRAIN_FAME_TIER = 2`）才开放；未解锁时地图「历练」按钮显示 🔒 与所需阶梯提示。开局阶段的名声/经验来源是拜访、切磋、悬赏与设施挑战——历练是这些手段打出声望后解锁的自由练级方式，而非起手可用的无限刷分入口。

**城池通用/特色功能**：📍拜访（本地已现身武将名录，复用既有武将详情弹窗）、🏋️历练、🏪集市（**本地货摊**：每城每游戏日按种子生成 4 件货品（名声达「略有耳闻」第 1 阶起为 5 件），价格按城市行情系数浮动——对马岛黑市固定 0.8，其余城按城名哈希稳定落在 0.90~1.20，行脚商队奇遇期间取更低者；已购摊位当日售罄，宿营翌日全城重新上货）、⚒️铁匠铺（**各城专精一类宝物**：锻专精类只需材料 5+30 金（名声达「名满天下」第 7 阶起再减为 4+20 金），其余类型 6+40 金，保底进度与宝物库锻造共享）、📋悬赏榜（每城 2~3 条，名声达「名动一国」第 5 阶时一次性为每城 +1 条永久空缺，见下节）、🏯特色设施（每城一项，见下节）。

**导航层级与返回逻辑**：天下地图为战役的顶层主界面（「新游戏」「继续游戏」均直接进入）；角色详情（原「角色扮演」主页，含六维加点/团队/装备）为地图下的子页，通过状态条「🎭 详情」按钮进入，其返回按钮固定回到地图；宝物库（仓库/商店/锻造）可从地图（集市/铁匠铺）或角色详情页两处进入，返回按钮通过 `homeBase` 机制智能回到发起它的那一层，而非固定某一处。所有战斗结算弹窗的「返回养成」类按钮统一改为调用 `goHome()`，同样遵循这一"回到发起处"的规则。
